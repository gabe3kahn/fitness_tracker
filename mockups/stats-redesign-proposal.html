Dynamic Cap — How It Works

Any stat reaches 90 → Cap: 100 → 200
Any stat reaches 180 → Cap: 200 → 300
Any stat reaches 270 → Cap: 300 → 400
Cap = 100 × ceil(maxStat / 90). Bars always normalize to cap. Your dominant stat pulls the ceiling up for everyone — gaps between stats stay legible at any scale.

Claimable Bonus Formula

A separate counter tracks cumulative SP earned per stat from workouts/puzzles. Each time the counter crosses the next threshold, one claimable bonus becomes available.
threshold(n) = 60 × n
Bonus #1 available after60 SP generated
Bonus #2 available after120 SP generated
Bonus #3 available after180 SP generated
Bonus #N available after60 × N SP generated
Independent of stat level. Tracked in a new stat_bonus_trackers table: (user, hero, stat) → total_sp_generated, claims_made. Unclaimed bonuses stack — a 180-min run generates 3 claimable endurance bonuses at once.
